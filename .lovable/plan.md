

## Conversation Attraction in Brain Chat — turn the public lobby into a learning surface

Today, Brain Chat lines are injected into the UQRC field as isolated bumps and Infinity replies are picked at random from five hard-coded strings. The field never learns who-talks-to-whom, and Infinity's "physics" tag is cosmetic.

This plan wires conversational adjacency directly into the field so consecutive prompts from different speakers gravitationally attract each other on the lattice — and Infinity's reply is selected as the candidate that **minimises curvature between** those two speakers' field signatures, plus seeded into the language learner.

---

### Behavior

Scenario from the request:

```
User A: "Hey hows it going"
Infinity: [Current Physics: q=0.42, basins=3, λ★=18]   ← already exists
User B: "Good how are you."
```

After this change:

1. When User A speaks, the field gets a Gaussian bump at A's lattice sites (already happens) **plus a soft pin tagged `lastSpeaker`**.
2. When User B speaks within the conversation window (≤90 s), B's bump amplitude is boosted along the **context axis** at A's sites, not just B's. This is the "attraction" — B's signal physically reinforces A's region of the lattice. Symmetrically A's residual is reinforced at B's sites.
3. The pair (A→B) is recorded by `languageLearner` as a high-trust transition (A's last sentence → B's first sentence), so Infinity actually learns the dialog flow, not just isolated tokens.
4. Infinity's reply, when triggered, is no longer random. It runs `selectByMinCurvature` over a candidate pool composed of:
   - the existing 5 stock physics-flavored lines (kept as a floor), **plus**
   - 3-5 candidates generated by `languageLearner.generate()` seeded from the merged tokens of the last A+B turn.
   The pick is the one that **lowers** curvature most when ghost-injected — i.e., the response that best bridges A and B in field space.
5. Infinity's `[Current Physics]` tag now reports the **delta** it just resolved (`Δq = -0.07, bridged @site 142`), giving users feedback that their conversation moved the field.

Net effect: the more two users converse in the lobby, the more their lattice regions overlap, the easier the language learner predicts continuations, and the better Infinity's bridging replies get. This is exactly the "self-stabilizing network through curvature minimisation" rule from workspace knowledge, applied to dialog.

### Implementation

**1. New helper `src/lib/uqrc/conversationAttraction.ts`**

Pure functions, no React:

- `recordTurn(speakerId: string, text: string, ts: number): TurnRecord` — pushes onto a 16-entry ring buffer of recent turns.
- `getPrevTurn(excludeSpeakerId: string, withinMs = 90_000): TurnRecord | null` — last turn from a *different* speaker inside the window.
- `attractToPrev(currentText: string, prev: TurnRecord, engine: FieldEngine): { dq: number; bridgeSite: number }`:
  - Inject `currentText` at its own sites with full amplitude (axis 0).
  - Inject `currentText` at `prev`'s lattice sites with **half** amplitude on the **context axis (axis 1)** — the literal "attract toward previous speaker."
  - Inject `prev.text` back at the current text's sites with quarter amplitude on axis 1 — symmetric pull.
  - Soft-pin the midpoint site between the two textSites centroids with stiffness 0.3 as `bridge:<a>:<b>` (capped, expires after 60 s via `pin(..., null)` cleanup).
  - Returns the resulting `Δq` and the chosen bridge site index.
- `selectBridgingReply(candidates: string[], prev: TurnRecord, current: TurnRecord, engine: FieldEngine): string` — wraps `selectByMinCurvature` but evaluates each candidate **after** the attractor injections are applied to the cloned ghost, so the chosen reply minimises curvature *given the bridge*, not the cold field.

**2. Wire it in `BrainUniverseScene.tsx` (`handleSend`, ~line 838)**

Replace the body of `handleSend` so that:

- After local broadcast and the existing `physics.injectAt(self.pos, …)` 3-D scene perturbation,
- Call `recordTurn(selfId, text, Date.now())`.
- Call `getPrevTurn(selfId)` → if present, call `attractToPrev(text, prev, engine)`. Use the returned `dq, bridgeSite` in the Infinity tag (see step 4).
- Feed the merged `${prev.text} ${text}` into `languageLearner.observeText(merged, { reward: 0.4, trust: 0.8 })` so the dialog edge is learned.
- When Infinity is called, build candidates = `[…stockPhysicsLines, …languageLearner.generate({ seed: lastTurnTokens, n: 4 })]` then `selectBridgingReply(candidates, prev, current, engine)`.

Also subscribe in the existing remote-chat `useEffect` (~line 821) so **remote** lines call `recordTurn(remote.peerId, remote.text, remote.ts)` — otherwise attraction only fires for the local user.

**3. Update Infinity's `[Current Physics]` line**

In the same handler, prepend the chosen reply with a compact tag that now includes the bridging metric:

```
[Δq=-0.072 · q=0.41 · ↔A↔B@s142]  the curvature near …
```

Falls back to the old `[q=…]`-only form when no `prev` exists (first speaker in a window).

**4. Cap & cleanup**

- Ring buffer holds last 16 turns globally (≈ 5 KB). On each new turn, prune entries older than 5 min.
- Bridge pins capped at 8 simultaneous (`engine.getPinCount()` check). LRU eviction releases the oldest `bridge:*` pin before adding a new one.
- All data is in-memory only — never serialized into mesh messages (Core memory rule: raw `u`/pins never broadcast).

**5. Tests**

New `src/lib/uqrc/__tests__/conversationAttraction.test.ts`:

- `recordTurn` ring-buffer eviction at 16 entries.
- `getPrevTurn` excludes same speaker and respects 90 s window.
- `attractToPrev` lowers Q at the bridge site after injection (compare curvatureMap before/after).
- `selectBridgingReply` returns a non-stock candidate when one strictly minimises Δq more than the stock lines, otherwise falls through to a stock line — verifying Infinity prefers learned bridges when they exist.
- Pin cap: after 9 distinct turn pairs, oldest `bridge:*` pin is released before the 9th is added.

### Acceptance

```text
1. Two users talking in /brain within 90 s of each other cause the field to
   show a measurable curvature decrease at the midpoint between their
   lattice sites — visible in Q_Score history (drops, doesn't spike).
2. Infinity's reply, when triggered, is the candidate with the lowest Δq
   when ghost-injected on top of the bridge — verified by unit test.
3. Infinity's `[Current Physics]` tag shows `Δq` and the bridge site index
   whenever a previous-speaker turn exists in the 90 s window; falls back
   to the existing q-only form otherwise.
4. The merged "previous turn + current turn" string is fed into the language
   learner with reward 0.4, trust 0.8, so dialog adjacency accumulates over
   sessions and is reflected in `LanguageSnapshot.transitionCount`.
5. No more than 8 `bridge:*` pins exist at once; oldest is released first.
6. Recent-turn ring buffer holds at most 16 entries and prunes >5 min old.
7. Raw field, pins, and ring-buffer data are never serialized into mesh
   messages — only existing chat lines travel the wire.
8. Existing tests pass; new tests cover ring-buffer, attractor, selector,
   and pin cap.
```
